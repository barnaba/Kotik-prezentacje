Dawniej...

HTML mógłby wyglądać tak:

1 <div class="todo ">
2   <a href="/todos/check/1"> invisible! </a>
3   <input class="check" type="checkbox">
4   <label class="todo-content">Przygotować prezentację</label>
5 </div>

Informacje o tym, jakie TODO użytkownik stworzył zapewne przechowalibyśmy w DOM-ie.

I po zaznaczeniu TODO jako wykonanego:

1 var todo_clicked = function(){
2   var url = $(this).parent().children('a').attr('href');
3   $.ajax({
4     url: url
5   });
6   ...
7 }

Na podstawie DOMu próbujemy dookoła wyciągać ID, zrobienie czegokolwiek wymaga trawersowania drzewa DOM

6/16

data-*

Ten wzorzec był do tego stopnia popularny, że w 2010 roku W3C rozszerzyło specyfikację HTML5 o:

3.2.3.8 Embedding custom non-visible data with the data-* attributes

1 <div class="todo">
2   <input class="check" type="checkbox" data-todo-id="1">
3   <label class="todo-content">Przygotować prezentację</label>
4 </div>

A funkcja JS:

1 var todo_clicked = function(){
2   $.ajax({
3     url: todo_path + $(this).attr('data-todo-id');
4   });
5   ...
6 }
